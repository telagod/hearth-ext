# Hearth — Personal Knowledge Companion

> Your reading, remembered. Quietly.
>
> 把你在网上读过的每一段值得回看的话，变成一团有温度、会主动找你的知识余烬。

---

## 0 · 一句话定位

**Hearth 是一款本地优先的浏览器插件，让"旧笔记主动找你"。**

不是 ChatGPT 套壳，不是 Notion 第二大脑，不是 Evernote 现代版——
是 **Readwise + Khoj 的开源轻量化形态，All-in 浏览器插件**，加一层「拟人化温度」。

---

## 1 · 立项动机

### 1.1 真痛点

| 现状 | 痛在哪 |
|---|---|
| Notion Web Clipper | 收完就忘，主界面打开率个位数 |
| Evernote / Pocket | 收藏夹坟场，2025 Pocket 已死 |
| Readwise | 闭源订阅，国内不可用，强绑 Kindle/RSS |
| AnythingLLM / Khoj | 需自托管，门槛高，浏览器体验弱 |
| Obsidian | 编辑器强，**捕捉环节缺失**，需另装 web clipper |

### 1.2 三个"反直觉"洞察

1. **用户不会主动打开知识库**——所以记录入口必须**寄生在浏览动作内**
2. **自动嗅探必死**——隐私墙 + 信噪比墙；记录权必须**握在用户的一秒决策里**
3. **检索不是终点，召回才是**——用户搜不出他不知道自己有的笔记；**让旧笔记主动找上门**比让用户搜索重要 10 倍

### 1.3 Hearth 的差异化命门

> **温度引擎（Warmth Engine）**

不是冷冰冰的"3 条相关结果"，而是：

```
☕ Hearth 想起来了：
   你 3 周前读 SQLite WAL 时也思考过这个并发问题，
   当时你的批注是「fsync 才是真瓶颈」。
   要看那段笔记吗？
```

LLM 不做"对话陪伴"，做"知识管家旁白"。这是 Hearth 和 AnythingLLM 的本质差。

## 4 · 四层切入：摩擦阶梯

> 用户在不同心力状态下都有合适入口；**全程不必主动打开主界面**。

### L0 · 寄生候选层（0 摩擦）

后台被动产生**候选信号**——只是候选，不算入库：

| 信号 | 触发 | 落地 |
|---|---|---|
| 文本复制 | `copy` 事件 | `inbox(kind=clip)` |
| 深度阅读 | 停留 > 60s + 滚动 > 50% + 选中 > 0 次 | `inbox(kind=read)` |
| 截图 | 浏览器截图 API + Tesseract OCR | `inbox(kind=image)` |

⚠ **隐私边界划在这条线上**：候选只存本地、72 小时未确认自动 TTL，不上云。

### L1 · 选中浮 bar 层（1 秒决策）—— **MVP 核心**

用户选中任意文本 → 浮一个 4 按钮迷你 bar：

```
┌────────────────────────────────────────────┐
│  📎 存  🤖 问  🔗 关联  💭 想  │
└────────────────────────────────────────────┘
```

| 按钮 | 行为 |
|---|---|
| 📎 存 | 一秒入库，自动取页面 meta + 上下文段落 |
| 🤖 问 | 在选中文本上跟 LLM 对话，结果可入库 |
| 🔗 关联 | 调温度引擎，吐"你库里 3 条相关" |
| 💭 想 | 加一句私人批注后入库（按 Tab 触发） |

### L2 · 反向召回层（0 摩擦，被动呈现）—— **差异化命门**

打开任意新页面 → 后台**轻量粗筛**库内潜在相关 → **LLM 精筛 + 拟人化**：

```
1) 提取当前页 title + 前 500 字 → 抽 5–8 个关键词
2) FTS5 在 highlights/notes 上召回 top 30
3) 用 SimHash + 关键词重合度，本地评分压到 top 5
4) 若分数 > 阈值，调 LLM 写"温度旁白"（仅 top 3 标题 + 当前页 title 给云，不发全文）
5) 角落浮一个小球 ☕，hover 展开
```

**Hearth 想起来了 ☕**——拟人化语气是产品 DNA，不是装饰。

### L3 · 周回顾层（5 分钟/周）—— **skill.md 用武之地**

每周一 09:00（用户可改），跑 `skills/weekly-review.md`：

1. 聚合上周 Inbox + 入库高亮
2. LLM 抽主题 → 3–5 张主题卡
3. sidepanel 弹"周回顾"，用户刷一遍：删 / 留 / 合并 / 改标签

## 5 · 温度引擎（Warmth Engine）

> 这是 Hearth 的灵魂。**不要做成"相似度搜索"。**

### 5.1 技术方案：**Hybrid Lite**

```
┌─────────────────────────────────────────┐
│ 1. Keyword Extract (本地, TF-IDF/RAKE)  │
│ 2. FTS5 BM25 召回 top 30                │
│ 3. SimHash 64bit 局部敏感哈希去重 + 评分│
│ 4. Recency Decay + 用户星标加权          │
│ 5. LLM Rerank+Narrate (top 3)           │
└─────────────────────────────────────────┘
```

**无本地嵌入模型**——FTS5 + SimHash 在数千条笔记规模下完全够用，包体压在 5MB 内。
当库 > 5000 条且 LLM API 不可用时，再提示用户选装 onnxruntime-web + bge-small（lazy install，~30MB）。

### 5.2 LLM 拟人化叙事

Prompt 模板（节选）：

```
你是 Hearth，用户的本地知识管家。
说话短、有温度，像在火炉边轻声提醒。

【当前页】<title>
【候选旧笔记】(top 3, 含标题/批注/日期)
1. ...
2. ...
3. ...

写一段不超过 60 字的旁白，语气：
- 用"你"称呼，不要"用户"
- 若批注里有用户原话，引用一句
- 不要列表，写成一句话两句话
- 不要恭维，不要"很好的问题"
```

### 5.3 边界

- LLM 只看 **当前页标题** + **候选笔记标题/批注**，**不发全文**
- LLM 调用前弹"明示气泡"，用户可关
- 离线时降级为纯统计："你库里有 3 条相关，要看吗？"

## 6 · 隐私模型

### 6.1 三层数据分级

| 级别 | 内容 | 出网规则 |
|---|---|---|
| **L0 候选** | Inbox 暂存（页面 meta、复制文本） | **绝不出网** |
| **L1 已入库** | 高亮、笔记、批注 | **默认不出网**；仅用户手动"问 AI"时按段发送 |
| **L2 元信息** | 标题、URL、关键词、SimHash | **可选出网**（温度引擎旁白生成） |

### 6.2 出网三戒

1. **明示** — 每次调外部 API 前，最近 24h 第一次需弹气泡确认
2. **最小** — 只发标题与必要片段，不发全文，除非用户在"问"按钮主动触发
3. **可控** — 设置面板列出"过去 7 天云端调用清单"，可一键清空

### 6.3 数据本地化与导出

| 导出格式 | 内容 |
|---|---|
| `hearth-export.zip` | 完整 SQLite + 全部 md + skills/*.md + 设置 |
| `obsidian-vault/` | 纯 md 文件夹，可直接做 Obsidian Vault |
| `hearth.json` | 单文件 JSON 备份（小库） |

导出在 **本地完成**，无任何后端中转。

## 7 · Office / 图片：输入源而非编辑器

**只做抽文入库，不做编辑器。**

| 类型 | 库 | 流程 |
|---|---|---|
| `.docx` | `mammoth.js` | 抽 HTML→md→入库 |
| `.pdf` | `pdf.js` | 抽文本+页号→入库；标注页码 |
| `.png/.jpg` | `tesseract.js` (按需 lazy load) | OCR→入库；缩略图存 IndexedDB |
| `.md` | 直读 | 入库 |
| `.csv/.xlsx` | `xlsx`（SheetJS）社区版 | 抽表头+预览→入库 |

⚠ **不做的**：
- ❌ 不做 docx/pptx 在插件内编辑（OnlyOffice 路线，包体爆炸）
- ❌ 不做图片美化/标注画板（与 MyMind/Eagle 路线打架）
- ❌ 不做 PDF 阅读器（pdf.js 仅作抽文工具）

## 8 · skill.md 自动化编排

### 8.1 skill 是什么

**一个 markdown 文件 + frontmatter，描述一个可被 Hearth 自动或手动触发的工作流。**

灵感来自 Anthropic Agent Skills，但**线性、可解释、无嵌套 Agent**。

```markdown
---
name: weekly-review
description: 每周一聚合上周高亮，生成 3-5 张主题卡
trigger:
  type: cron
  schedule: "0 9 * * MON"
tools: [db.query, llm.summarize, ui.notify]
permissions:
  llm: required
  network: optional
---

# 步骤

1. 查询上周入库的 highlights 与 notes
2. 调 LLM 抽 3-5 个主题
3. 每主题生成一张卡片（标题 + 摘要 + 链接）
4. 通过 ui.notify 推送到 sidepanel
```

### 8.2 内置 skills（M1）

| skill | 触发 | 作用 |
|---|---|---|
| `inbox-tidy` | cron 每日 22:00 | 候选清单 LLM 打标签，72h 未确认 TTL |
| `weekly-review` | cron 每周一 09:00 | 上周回顾，主题卡 |
| `tag-suggest` | 入库时触发 | 自动建议 3 个标签 |
| `link-similar` | 入库后异步 | 寻找库内相关并自动加 `[[wiki link]]` |
| `monthly-purge` | cron 每月 1 号 | 30 天未访问 + 0 星标的入库笔记打"冷藏" |

### 8.3 沙箱

skill 运行环境：
- **白名单工具**：只能调注册过的 tool（db.query 限只读，llm.* 受限于设置）
- **不能直接读 DOM**：所有数据从 db.* 取
- **超时**：30s 硬上限
- **审计日志**：每次运行落 `skill_runs` 表

## 9 · 信息架构

```
┌────────────────────────────────────────────────────┐
│  Hearth Sidepanel                                 │
│                                                    │
│  ┌──────┬──────┬──────┬──────┬──────┐           │
│  │ 库   │ 对话 │Inbox │Skills│设置 │             │
│  └──────┴──────┴──────┴──────┴──────┘           │
│                                                    │
│  ┌────────────────────────────────────────────┐  │
│  │  当前 tab 反向召回卡片（如果命中）         │  │
│  └────────────────────────────────────────────┘  │
│                                                    │
│  ┌────────────────────────────────────────────┐  │
│  │  最近入库 / 搜索 / 标签云 / 时间线         │  │
│  └────────────────────────────────────────────┘  │
└────────────────────────────────────────────────────┘
```

- **库（Library）**：检索 + 浏览 + 时间线 + 标签云
- **对话（Chat）**：以选中内容/笔记为上下文的 LLM 对话；可保存为笔记
- **Inbox**：候选清单 + 一键转入库/丢弃
- **Skills**：列表 + 启停 + 历史日志 + 编辑
- **设置**：API keys、隐私规则、导出、外观

另：**Newtab Dashboard**（可选启用）—— 打开新标签页时展示「今日记忆 / 本周回顾 / 高亮日历」。
